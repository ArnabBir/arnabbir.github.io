02. LSM Engine & Read Path

Memtable Structure

The Memtable is an in-memory sorted buffer (essentially a mutable sorted map, often a SkipList). It serves as the write buffer. To support concurrent reads and writes without blocking, Bigtable uses a Copy-on-Write mechanism for row rows. When the memtable fills up, it is frozen and converted into an immutable SSTable (Sorted String Table) file on GFS.

Two-Level Caching

Scan Cache

High-level cache storing Key-Value pairs returned by the SSTable interface. Great for repeated reads of same data.

Block Cache

Low-level cache storing raw 64KB SSTable blocks fetched from GFS. Great for spatial locality (sequential reads).

The Merged View

Since data is split across the Memtable and multiple SSTables, a read operation must execute on a merged view of all these sources. It uses a heap to merge iterators from the Memtable and SSTables, ensuring the most recent version (highest timestamp) of a key is returned.

03. Refinements

Locality Groups

Clients can group column families. A separate SSTable is generated for each group. Enables reading metadata without reading page contents.

Bloom Filters

Drastically reduces disk accesses. Allows server to ask if an SSTable might contain data for a row/column before reading.

Compression

Applied to each SSTable block. Bentley-McIlroy algorithm compresses common strings across a large window (great for web pages from same host).
